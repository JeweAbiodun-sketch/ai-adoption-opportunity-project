# Timeline Estimate — AI Demand Forecasting Implementation
**Mid-Market Retail · Conservative estimate**

---

## Summary Timeline

| Phase | Duration | Key Milestone |
|---|---|---|
| Phase 0 — Data audit | 2 weeks | Data readiness confirmed or gaps identified |
| Phase 1 — Vendor selection | 4 weeks | Vendor selected, free trial running |
| Phase 2 — Pilot deployment | 10 weeks | AI running alongside manual forecasts |
| Phase 3 — Evaluation | 4 weeks | Go/no-go decision with evidence |
| Phase 4 — Full rollout | 16 weeks | AI forecasting across all SKUs and stores |
| **Total to full implementation** | **~9 months** | |

---

## Realistic Expectations for Cleo

- **Month 1–2:** No visible results — data audit and setup phase
- **Month 3–4:** First AI forecasts generated; comparing to buyer estimates
- **Month 5–6:** First measurable results — stockout rate and overstock flags visible
- **Month 7–9:** ROI becomes visible in inventory cost and markdown rate
- **Month 12:** Full implementation ROI measurable against baseline

**The most common mistake:** Expecting results in month 1. Retail AI requires a full seasonal cycle to learn patterns properly. Set expectations with Cleo accordingly.

---

## Accelerators (if data is already clean)
- Clean ERP data reduces Phase 0 to 3–5 days
- Vendor with pre-built ERP connector reduces Phase 1 integration to 1–2 days
- Minimum viable timeline (best case, clean data): **4–5 months to measurable results**
